# Autonoe Planning Notes

## Session: 2026-02-21 (EXT-002 completion)

### Accomplished This Session

Implemented EXT-002: MutationObserver for dynamically loaded book entries.

Changes to `content.js`:
- Added `processAddedNodes(addedNodes)` — iterates a NodeList of added DOM nodes,
  checks each element node both directly (id startsWith 'bw_buy_') and via
  `querySelectorAll('div[id^="bw_buy_"]')` for descendants, then calls
  `processContainer()` on each match.
- Added `observeDynamicContent()` — creates a `MutationObserver` watching
  `document.body` with `{ childList: true, subtree: true }`, calls
  `processAddedNodes()` for each mutation's `addedNodes`, and registers a
  one-time `pagehide` listener that calls `observer.disconnect()` for cleanup.
- Entry point now calls `observeDynamicContent()` after `processAllContainers()`.

Changes to `test.js`:
- Upgraded `test()` runner to support async (Promise-returning) test functions;
  wraps all tests in a top-level `async` IIFE with `await` before each `test()`.
- Added helper `flushMicrotasks()` (`Promise.resolve()`) to drain JSDOM's
  MutationObserver microtask queue in async tests.
- Added 6 new MutationObserver tests (section: `[ MutationObserver — dynamic content ]`):
  1. Badge appears on bw_buy_ div added dynamically to body
  2. Badge appears when a parent element containing bw_buy_ is added dynamically
  3. No duplicate badge when bw_buy_ node already has a badge before observer fires
  4. No badge added dynamically for book with 0% discount
  5. MutationObserver does not activate on non-wishlist pages
  6. Multiple dynamic entries added in one mutation batch are all processed
